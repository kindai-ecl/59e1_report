# 卒業研究計画書

## マルチモーダル学習による薬剤鑑別システムの構築

近畿大学 情報学部 / 2026年度

---

## 1. 研究テーマ

**Qwen3.5 を用いたマルチモーダル学習による薬剤鑑別システムの構築**

錠剤の外観画像と薬剤名・効能・形状などのテキスト情報を組み合わせて、オープンソースLLM（Qwen3.5）に学習させることで、薬剤鑑別システムを構築する。提案手法として「RAGによる検索拡張型」と「ファインチューニング型」の2種類を実装し、どちらがより有効かを比較する。

### Qwen3.5 を選んだ理由

- オープンソースのため無料でファインチューニングでき、患者データをAPIに送る必要がないのでプライバシーのリスクがない
- 画像とテキストを同時に入力して一括で学習できるため、複雑な処理工程が不要
- ChatGPT・Geminiとの精度比較により、提案手法の優位性を客観的に示せる

---

## 2. 従来手法と問題点

本研究では以下3本の論文を参考文献とし、それぞれの限界から提案手法の必要性を示す。

### ① 画像のみの手法

**ePillID Dataset: A Low-Shot Fine-Grained Benchmark for Pill Identification**
Usuyama et al. / CVPR Workshop 2020
URL: https://arxiv.org/abs/2005.14288

#### 手法概要

ディープラーニング（ResNet）と類似度学習を使い、**9,804種類の錠剤を1種類あたり参照画像1枚だけ**で識別するベンチマークを構築・評価した。参照画像が1枚しかないという厳しい条件が、画像のみで識別することの限界を明確に示している。

#### 問題点

- 参照画像が1枚しかないため、見た目が似ている薬の識別精度が大きく落ちる
- 白い丸い錠剤や同じ成分で用量だけ違う薬など、外観がほぼ同じ薬を見分けられない
- 薬の名前・効能・刻印文字などのテキスト情報をまったく使っていないため、識別の手がかりが少ない
- 米国のデータセットのため、日本の薬や日本語のテキストには対応できない

---

### ② 刻印文字の読み取り（OCR）による手法

**GO-PILL: A Geometry-Aware OCR Pipeline for Reliable Recognition of Debossed and Curved Pill Imprints**
MDPI Mathematics 2025
URL: https://www.mdpi.com/2227-7390/14/2/356

#### 手法概要

錠剤表面の曲面や凹み文字を考慮した画像補正をかけたうえでOCR（文字認識）を行い、読み取った文字を薬剤データベースと照合して識別する。Top-1 F1スコア: 81.83%。

#### 問題点

- 刻印のない錠剤・カプセルには使えない
- 文字を読み取った後に薬剤データベースと照合する必要があり、データベースに載っていない薬には対応できない
- 色・形・大きさなどの見た目の特徴とテキスト情報を組み合わせる仕組みがない
- 刻印の摩耗・光の反射・撮影角度によって認識精度が大きく変わる

---

### ③ LLMによるゼロショット識別手法

**Identification of Kidney-related Medications Using AI from Self-captured Pill Images**
Saad et al. / PMC 2024
URL: https://www.ncbi.nlm.nih.gov/pmc/articles/PMC11391868/

#### 手法概要

GPT-4に25種類の腎臓科の処方薬の写真を見せて識別させた実験。22/25（88%）を正解したが、刻印文字の読み間違いが主な失敗原因だった。

#### 問題点

- APIを通じてしか使えないためファインチューニングができず、特定の薬剤に特化した学習ができない
- 患者の錠剤写真を外部APIに送信するためプライバシー上のリスクがある
- 特定分野の薬剤に対する学習データが不足しており、専門領域への対応が弱い
- 事前学習のみ（ゼロショット）のため、見た目が似た薬の細かい識別に限界がある
- 利用コストが高く、病院・薬局での継続的な運用が難しい

---

## 3. テーマの問題点と解決方法

### 現状の問題

- **画像のみの手法**：見た目が似た薬の識別で精度が頭打ちになる
- **OCR手法**：刻印のない薬には使えず、データベース照合に依存している
- **LLM（ChatGPT等）**：ファインチューニング不可・専門領域対応が弱い・プライバシーリスク・高コスト

### 解決方法：2つの提案手法

オープンソースLLMであるQwen3.5をベースに、アプローチの異なる2つの手法を実装して比較する。

#### 提案手法① Qwen3.5 + RAG（検索拡張生成）

薬剤データベースをあらかじめベクトル化しておき、入力された画像・テキストに似た薬剤情報を自動で検索してQwen3.5に渡すことで識別精度を高める手法。

**強み**

- 新しい薬が追加されてもデータベースを更新するだけでよく、再学習が不要
- 少ないデータでも運用できる
- 「この薬に似ている」という識別の根拠を提示できる

**課題**

- 検索の精度がそのまま識別精度のボトルネックになる
- 識別のたびにデータベース検索が走るため、処理に時間がかかる

#### 提案手法② Qwen3.5 + ファインチューニング（FT）

実薬データ（錠剤画像＋薬剤テキスト）でQwen3.5を追加学習し、薬剤鑑別に特化したモデルを構築する手法。

**強み**

- 薬剤鑑別に特化した学習による高い識別精度
- 推論が速く、完全にローカル環境で動かせる
- 外部APIを使わないのでプライバシーリスクがない

**課題**

- 新しい薬が追加されるたびに再学習が必要
- ある程度の量の学習データが必要

## 4. 評価方法

### 比較手法（4条件）

| 条件          | 手法                    | 位置づけ |
| ------------- | ----------------------- | -------- |
| Baseline①     | ChatGPT（ゼロショット） | 比較対象 |
| Baseline②     | Gemini（ゼロショット）  | 比較対象 |
| **提案手法①** | **Qwen3.5 + RAG**       | 提案手法 |
| **提案手法②** | **Qwen3.5 + FT**        | 提案手法 |

### 評価指標

- **Top-1 正解率**：AIが「これだ」と1番自信を持って答えた薬が正解だった割合
- **Top-5 正解率**：AIが挙げた上位5候補の中に正解が含まれていた割合

### Ablation Study（どの情報が識別に効いているかを確かめる実験）

「もし画像だけだったら？」「テキストだけだったら？」と条件を変えた実験を行い、画像とテキストを組み合わせることの効果を数字で示す。

| 条件                   | 画像入力 | テキスト入力 |
| ---------------------- | -------- | ------------ |
| 画像のみ               | あり     | なし         |
| テキストのみ           | なし     | あり         |
| マルチモーダル（提案） | あり     | あり         |

→ それぞれの精度を比較することで、マルチモーダル（画像＋テキスト）の有効性を証明する。
