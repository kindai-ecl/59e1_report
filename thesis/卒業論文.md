# 卒業研究計画書

## マルチモーダル学習による薬剤鑑別システムの構築

近畿大学 情報学部 / 2026年度

---

## 1. 研究テーマ

**Qwen3.5 を用いたマルチモーダル学習による薬剤鑑別システムの構築**

錠剤の外観画像と薬剤名・効能・形状などのテキスト情報を組み合わせて、オープンソースLLM（Qwen3.5）に学習させることで、薬剤鑑別システムを構築する。提案手法として「RAGによる検索拡張型」と「ファインチューニング型」の2種類を実装し、どちらがより有効かを比較する。

### Qwen3.5 を選んだ理由

- オープンソースのため無料でファインチューニングでき、患者データをAPIに送る必要がないのでプライバシーのリスクがない
- 画像とテキストを同時に入力して一括で学習できるため、複雑な処理工程が不要
- ChatGPT・Geminiとの精度比較により、提案手法の優位性を客観的に示せる

---

## 2. 従来手法と問題点

本研究では以下3本の論文を参考文献とし、それぞれの限界から提案手法の必要性を示す。

### ① パイプライン型マルチモーダル識別手法

**An Accurate Deep Learning-Based System for Automatic Pill Identification: Model Development and Validation**
Heo et al. / Journal of Medical Internet Research 2023
DOI: 10.2196/41043

#### 手法概要

YOLOv5で錠剤表面の刻印文字を検出し、ResNet-32で形状・色・形態（錠剤/カプセル）を認識する。さらに、RNNベースの文字レベル言語モデルで刻印文字を補正し、薬剤データベースとの照合で最終識別を行うパイプライン型システム。韓国データセットTop-1精度85.6%、米国データセット74.5%を達成。

#### 問題点

- 刻印のないカプセルや素錠には適用できない
- 各モジュールが独立しており、誤差が積み上がる
- 日本語薬剤への対応が示されていない
- 新薬追加時に識別モジュールの再学習が必要となる場合がある

---

### ② マルチモーダルメトリクス学習による識別手法

**Metric-based Pill Recognition with the Help of Textual and Visual Cues**
Rádli et al. / IET Image Processing 2024
DOI: 10.1049/ipr2.13273

#### 手法概要

EfficientNetV2をバックボーンにRGB・LBP・輪郭・テクスチャの4ストリームで視覚特徴を抽出し、薬剤説明テキストから動的マージントリプレット損失（DMTL）のマージンを決定するメトリクス学習フレームワーク。112クラス・4,480枚のOGYEIv2データセットでTop-1精度93.56%を達成。

#### 問題点

- 評価対象が112クラスの小規模データセットに限定されており、大規模化への対応が未検証
- ハンガリー語データセットのため、日本語薬剤への応用が示されていない
- LLMを使用していないため、端から端まで一体的な学習ができない
- 新薬追加のたびにメトリクス学習の再学習が必要

---

### ③ LLMによるゼロショット識別手法

**Drug Pills Identification System using Google Gemini LLM: A Generative AI Approach**
Menaha R., Abilaash R., Mohanram N. P., Akash Unnikrishnan, Sukumar S.
ICERCS 2024 / DOI: 10.1109/ICERCS63125.2024.10895371

#### 手法概要

Google Gemini LLMを用いて錠剤画像から薬剤情報を提供するシステム。画像をバイナリ値に変換し、オントロジーベースのプロンプトエンジニアリングとともにGeminiに入力する。807枚の薬剤画像に対してTop-1精度95.04%を達成。

#### 問題点

- 商用クローズドソースモデルのため、特定薬剤へのファインチューニングが不可能
- 患者の薬剤画像を外部APIに送信するため、プライバシー上のリスクがある
- 利用コストが高く、病院・薬局での継続的な運用が困難
- ゼロショットのため、外観が類似した薬剤の細かい識別に限界がある

## 3. テーマの問題点と解決方法

### 現状の問題

- **パイプライン型（Heo 2023）**：刻印のない薬に使えず、モジュール分離で誤差が積み上がる
- **メトリクス学習型（Rádli 2024）**：小規模・非LLM・日本語非対応
- **LLM（Gemini等）**：ファインチューニング不可・プライバシーリスク・高コスト

### なぜマルチモーダルLLMが解決策になるか

錠剤の外観画像と薬剤名・形状・効能などのテキスト情報を同時に入力できるマルチモーダルLLMを用いれば、刻印がない薬剤であっても色・形・大きさなどの視覚的特徴とテキスト情報を組み合わせて識別できる。さらにオープンソースモデルであれば患者データを外部に送信せずローカルで動作させることができ、ファインチューニングによって特定薬剤への適応も可能である。

### 解決方法：2つの提案手法

オープンソースLLMであるQwen3.5をベースに、アプローチの異なる2つの手法を実装して比較する。

#### 提案手法① Qwen3.5 + RAG（検索拡張生成）

薬剤データベースをあらかじめベクトル化しておき、入力された画像・テキストに似た薬剤情報を自動で検索してQwen3.5に渡すことで識別精度を高める手法。

**強み**

- 新しい薬が追加されてもデータベースを更新するだけでよく、再学習が不要
- 少ないデータでも運用できる
- 「この薬に似ている」という識別の根拠を提示できる

**課題**

- 検索の精度がそのまま識別精度のボトルネックになる
- 識別のたびにデータベース検索が走るため、処理に時間がかかる

#### 提案手法② Qwen3.5 + ファインチューニング（FT）

実薬データ（錠剤画像＋薬剤テキスト）でQwen3.5を追加学習し、薬剤鑑別に特化したモデルを構築する手法。

**強み**

- 薬剤鑑別に特化した学習による高い識別精度
- 推論が速く、完全にローカル環境で動かせる
- 外部APIを使わないのでプライバシーリスクがない

**課題**

- 新しい薬が追加されるたびに再チューニングが必要
- ある程度の量の学習データが必要

## 4. 評価方法

### 比較手法（4条件）

| 条件          | 手法                    | 位置づけ |
| ------------- | ----------------------- | -------- |
| Baseline①     | ChatGPT（ゼロショット） | 比較対象 |
| Baseline②     | Gemini（ゼロショット）  | 比較対象 |
| **提案手法①** | **Qwen3.5 + RAG**       | 提案手法 |
| **提案手法②** | **Qwen3.5 + FT**        | 提案手法 |

### 評価指標

- **Top-1 正解率**：AIが「これだ」と1番自信を持って答えた薬が正解だった割合
- **Top-5 正解率**：AIが挙げた上位5候補の中に正解が含まれていた割合

### Ablation Study（どの情報が識別に効いているかを確かめる実験）

「もし画像だけだったら？」「テキストだけだったら？」と条件を変えた実験を行い、画像とテキストを組み合わせることの効果を数字で示す。

| 条件                   | 画像入力 | テキスト入力 |
| ---------------------- | -------- | ------------ |
| 画像のみ               | あり     | なし         |
| テキストのみ           | なし     | あり         |
| マルチモーダル（提案） | あり     | あり         |

→ それぞれの精度を比較することで、マルチモーダル（画像＋テキスト）の有効性を証明する。
